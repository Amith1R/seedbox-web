TS
Subtitle Timing Shift
Shift every .srt in a folder, keep backups, and track the work as a reusable job.
This processes all top-level .srt files in the chosen folder. Each changed file gets a .bak backup before timing is updated.
ST
Subtitle Convert / Match
Convert subtitle formats to .srt and rename subtitle files to match video files in a folder.
`Convert` scans for `.ass`, `.ssa`, `.vtt`, and `.sub` files and creates/updates `.srt` files. `Rename / Match` tries to align subtitle filenames with the video files in the same folder.
MR
Media Rename Cleanup
Normalize separators like dots and underscores for cleaner media filenames.
MO
Media Move / Organize
Move files from a source folder into a destination, either flat or grouped into folders per video.
JF
Jellyfin Library Rescan
Trigger a Jellyfin library refresh after organizing or updating media.
BR
Bulk Rename
Find and replace text in filenames, with optional folder renaming and recursive scanning.
SC
Storage Reports & Cleanup
Run duplicate scans, largest-file reports, folder-size analysis, orphan subtitle checks, and empty-folder cleanup from one card.
`Largest files` and `Folder sizes` use the limit value. `Duplicates`, `Orphan subtitles`, and `Empty folders` use the same folder and recursive setting.
AR
Archive, Extract & Backup
Pack folders, extract archive files, or create dated backup archives without leaving the dashboard.
`Archive` expects a source folder. `Extract` expects a source archive file path. `Backup` creates a dated archive of the source folder in the destination.
MC
Temp & Log Cleanup
Clear temporary clutter, stale partials, cache leftovers, and old log files with tracked queue jobs.
Temp cleanup removes things like `.tmp`, `.part`, `.crdownload`, `.aria2`, `__pycache__`, `.DS_Store`, and `Thumbs.db`. Log cleanup removes log-like files older than the threshold.
RP
System Reports
Generate system and disk reports you can keep, compare, or share without opening the terminal.
System reports include health, uptime, IP, top processes, job history, mounts, and Docker state. Disk snapshots focus on folder sizes, mounts, and storage usage.
